## 一、问题引入（~30 秒）

> 各位老师好，我今天汇报的课题是**基于 YOLOv8 的口罩佩戴检测系统**。
>
> 疫情以来，公共场所的口罩佩戴管理从临时措施变成了长期刚需。传统的人工检查有三个痛点：效率低、容易漏检、存在交叉感染风险。所以我想做一个基于深度学习的自动检测系统——输入一张图片或视频流，输出检测到的人脸位置和是否佩戴口罩的判断。
>
> 选题的工程意义在于：这是一个完整的目标检测流水线，从数据处理、模型训练到评估分析，覆盖了深度学习工程实践的全流程。

## 二、技术方案（~40 秒）

> 在模型选型上我选了 **YOLOv8n**。为什么是它？三个原因：
> 1. YOLO 系列是单阶段检测器，速度和精度的平衡最好，适合实时场景。
> 2. v8 相比之前的版本，用了 Anchor-Free 检测头和解耦分类/回归分支，精度更高。
> 3. Nano 版本只有 300 万参数、8.1 GFLOPs，在 RTX 4060 上单张推理约 3 毫秒。
>
> 整体系统分为四个阶段，恰好对应我的四个脚本：
>
> | 阶段 | 脚本 | 职责 |
> |------|------|------|
> | 数据准备 | `prepare_data.py` | 合并数据集、过滤清洗、划分训练/验证/测试集 |
> | 模型训练 | `train_yolo.py` | 加载预训练权重、配置增强策略、微调训练 |
> | 模型评估 | `evaluate_yolo.py` | 计算 mAP/精确率/召回率、生成可视化和检测样本 |
> | 报告分析 | `report_analysis.py` | 补充统计图表、置信度分析、生成实验报告 |

## 三、数据准备 — 代码讲解（~60 秒）⚡

> 先看数据预处理这块。这是整个项目的基础，数据处理的质量直接决定了模型的上限。
>
> 核心代码在 `prepare_data.py`，我讲三个关键设计：

### 3.1 样本收集与过滤

```python
def collect_samples(source_dirs, skip_empty=True):
    samples = []
    for src_dir in source_dirs:
        for jpg in sorted(src.glob("*.jpg")):
            txt = jpg.with_suffix(".txt")        # 同名 txt 标注文件
            lines = [l for l in txt.read_text().strip().split("\n") if l.strip()]
            if skip_empty and not lines:          # 过滤空标注
                continue
            samples.append({"image": jpg, "label": txt, "lines": lines, ...})
    return samples
```

> **设计要点：**
> - 图片和标注用**同名不同后缀**匹配（`a.jpg` ↔ `a.txt`），这是 YOLO 格式的标准惯例。
> - 过滤掉空标注文件——这些图片里没有人脸目标，对训练没有贡献，留着反而会干扰模型。
> - 为了合并两个数据源且避免重名冲突，文件名加上了来源目录前缀（`all_mask_xxx.jpg`）。

### 3.2 类别分布统计

```python
class_counter = Counter()
box_counter = Counter()
for s in samples:
    for line in s["lines"]:
        cls_id = int(parts[0])
        box_counter[cls_id] += 1
```

> 在划分之前先统计类别分布，这是一个好习惯——能及早发现数据不均衡问题。结果发现 without_mask 约 1.5 万个框，with_mask 约 0.7 万个，大约是 2:1 的不均衡。这个发现对后面分析模型性能很有帮助。

### 3.3 随机划分与 data.yaml 生成

```python
random.shuffle(samples)
train_end = int(n * 0.70)
val_end   = train_end + int(n * 0.20)

splits = {
    "train": samples[:train_end],
    "val":   samples[train_end:val_end],
    "test":  samples[val_end:],
}
```

> 用 `random.seed(42)` 固定随机种子，确保每次运行划分结果一致，保证实验可复现。划分后按 YOLO 要求的目录结构复制图片和标注，最后自动生成 `data.yaml` 配置文件，里面包含了数据集路径和类别名称。

## 五、模型训练 — 代码讲解（~90 秒）⚡

> 训练脚本 `train_yolo.py` 是整个项目的核心。我用 Ultralytics 框架来训练，但关键参数的选取和增强策略的配置需要自己设计。

### 5.1 模型加载与多模型对比设计

```python
for model_name in args.model:       # 支持一次训练多个模型，方便对比
    model = YOLO(model_name)        # 自动下载预训练权重
    results = model.train(
        data=args.data,
        epochs=args.epochs,
        batch=args.batch,
        ...
        name=f"mask_detect_{model_display}",
    )
```

> **设计要点：**
> - 命令行支持传入多个模型（`--model yolov8n.pt yolov8s.pt`），一次运行完成多模型对比实验，省去反复改代码的麻烦。
> - 训练完自动汇总对比表格，打印每个模型的 mAP@50、mAP@50-95、精确率和召回率，一目了然。

### 5.2 数据增强策略

```python
model.train(
    ...
    hsv_h=0.015,         # 色调扰动 — 模拟不同光照色温
    hsv_s=0.7,           # 饱和度扰动 — 模拟不同摄像头色彩
    hsv_v=0.4,           # 亮度扰动 — 模拟明暗变化
    degrees=10.0,        # 旋转 ±10°
    translate=0.1,       # 平移 ±10%
    scale=0.5,           # 缩放 50%~150%
    fliplr=0.5,          # 50% 概率水平翻转
    mosaic=1.0,          # Mosaic 拼接增强
    mixup=0.1,           # MixUp 混合增强
    close_mosaic=10,     # 最后 10 轮关闭 Mosaic
)
```

> 这里我重点讲三个增强：
>
> **HSV 色彩扰动**：真实场景中摄像头色温、光照差异很大，HSV 扰动让模型对颜色变化更鲁棒。色调调得比较小（±0.015），因为人脸肤色是重要的区分特征，不能扰动太大。
>
> **Mosaic 增强**：把 4 张图随机缩放后拼成 1 张 640×640 的训练图。这样做的好处是把大目标缩小，增加了小目标样本量，提升了模型对小尺寸人脸的检测能力。但 Mosaic 拼接图的数据分布和真实图有差异——拼接图的背景是碎片化的，真实图是完整的。
>
> **所以有了 `close_mosaic=10`**——最后 10 个 epoch 关闭 Mosaic，让模型在真实数据分布上做最后的精细调整。这是 YOLOv8 社区实践出来的经验参数。

### 5.3 优化器与学习率策略

```python
model.train(
    ...
    lr0=1e-3,            # 初始学习率
    optimizer="auto",    # 自动选择 AdamW
    cos_lr=True,         # 余弦退火调度
    patience=15,         # 早停耐心值
)
```

> **为什么选 AdamW？** AdamW 在 Adam 的基础上把权重衰减和梯度更新解耦，泛化性能更好。配合余弦退火，学习率从 1e-3 平滑衰减到 1e-6，相比阶梯式衰减能让模型收敛到更优的局部最优点。
>
> **早停 patience=15**：连续 15 个 epoch 验证集指标没有提升就自动停止。实际训练在第 83 轮触发了早停，省下了 17 个 epoch 的计算时间。

## 九、报告分析 — 代码讲解（~40 秒）

> `report_analysis.py` 是评估之后的一个补充分析脚本，生成四类图表。我挑两个有代表性的讲。

### 9.1 置信度分布分析

```python
def plot_confidence_distribution():
    model = YOLO("best.pt")
    for img_path in tqdm(samples):
        results = model(str(img_path), conf=0.001)  # 极低阈值，收集所有预测
        # 遍历每个预测框，与真实标注计算 IoU
        for pi in range(len(pred_cls)):
            # 找到最佳匹配的真实框
            best_iou = max(iou for matching gt)
            is_correct = best_iou >= 0.45
            all_confidences.append((conf, is_correct, cls))
```

> **设计要点**：把置信度阈值降到 0.001，目的是收集模型的所有输出，包括那些置信度很低的"不确定"预测。然后逐一和真实标注计算 IoU——IoU ≥ 0.45 且类别匹配的算真正例，否则算假正例。这样我们就能分析：模型在什么置信度下是靠谱的，什么置信度下不靠谱。

### 9.2 IoU 计算

```python
def _compute_iou(pred_xywh, gx, gy, gw, gh):
    px1, py1 = pred_xywh[0] - pred_xywh[2]/2, pred_xywh[1] - pred_xywh[3]/2
    px2, py2 = pred_xywh[0] + pred_xywh[2]/2, pred_xywh[1] + pred_xywh[3]/2
    # ...ground truth 同样转成 (x1,y1,x2,y2)
    inter = max(0, ix2 - ix1) * max(0, iy2 - iy1)
    union = area_p + area_g - inter
    return inter / union if union > 0 else 0
```

> 这个 IoU 函数展示了一个重要的细节：YOLO 格式的标注是归一化的中心点坐标 `(x, y, w, h)`，计算 IoU 时需要先转换成左上角和右下角坐标 `(x1, y1, x2, y2)`。虽然逻辑简单，但写错一步就全错了，所以在注释里把坐标变换写清楚是很重要的编码习惯。

## 十二、项目工程亮点总结 📎

> 除了模型本身，我想强调这个项目的几个工程实践亮点：

### 12.1 命令行接口设计

> 四个脚本全部使用 `argparse` 提供命令行接口，关键参数全部可配置。比如训练脚本支持：
>
> ```bash
> python scripts/train_yolo.py --model yolov8n.pt --epochs 100 --batch 16 --lr 1e-3
> ```
>
> 这意味着换一个数据集、换一个模型、调整参数，都不需要改代码，一行命令搞定。这对于实验迭代和结果复现非常重要。

### 12.2 模块化设计

> 数据准备、训练、评估、报告分析四个环节各自独立，每个脚本可以单独运行。这样设计有两个好处：一是出问题时容易定位，二是中间产物（如划分好的数据集、训练好的权重）可以复用，不需要每次都从头跑。

### 12.3 可复现性保障

> - `random.seed(42)` 固定随机种子
> - `exist_ok=True` 支持断点续跑
> - `--resume` 支持从 checkpoint 恢复训练
> - `data.yaml` 和 `args.yaml` 自动记录所有配置参数
>
> 任何一个拿到这个代码的人，只要配好环境，一行命令就能完整复现我的实验结果。

|---|---|
| **"为什么选 YOLOv8 而不是其他模型？"** | YOLOv8 是当前主流框架，性能好、文档全。相比 Faster R-CNN 两阶段模型，YOLO 单阶段检测速度更快，适合实时场景。相比 YOLOv5，v8 使用了 Anchor-Free 检测头和解耦头，精度更高。 |
| **"mAP@50 和 mAP@50-95 有什么区别？"** | mAP@50 要求预测框和真实框的 IoU ≥ 0.5 就算对，比较宽松。mAP@50-95 在 0.5 到 0.95 之间每隔 0.05 取一个阈值然后平均，严格得多。我们的 mAP@50 是 0.758，mAP@50-95 是 0.511，差距说明定位精度还有提升空间。 |
| **"数据增强做了什么？为什么 Mosaic 最后 10 轮要关？"** | 用了 Mosaic、MixUp、HSV 扰动、随机翻转/缩放/旋转。Mosaic 把 4 张图拼成 1 张，增加了小目标样本但改变了数据分布，所以最后 10 轮关闭让模型在真实分布上做精细调整。这是 YOLOv8 社区的标准做法。 |
| **"代码里 `close_mosaic=10` 是什么意思？"** | 训练的最后 10 个 epoch 关闭 Mosaic 数据增强。epoch 90 之后模型看的是未经拼接的真实图片，让模型在部署时的真实数据分布上做最后的 fine-tune，有利于提升最终精度。 |
| **"你的类别不均衡怎么处理的？"** | 当前没有做专门的不均衡处理，因为 2:1 的比例在目标检测中还算可接受。如果要优化，可以考虑：对少数类过采样、Focal Loss 给少数类更高权重、或者采集更多戴口罩的数据。 |
| **"如果有密集人群场景怎么办？"** | YOLOv8 本身的检测能力可以应对中等密度人群。极端密集场景的瓶颈在于 NMS（非极大值抑制）可能把相邻的人脸误合并。可以考虑用 Soft-NMS 替代标准 NMS，或者换成专门的人脸检测模型如 RetinaFace 做级联。 |
| **"这个系统能实际部署吗？"** | 推理速度足够，可以接入 OpenCV 读取摄像头做实时检测。用 TensorRT 或 ONNX 导出后还能再加速 2-3 倍。主要挑战是真实场景中的极端光照、密集人群和遮挡，需要在实际环境中进一步测试迭代。 |
| **"`exist_ok=True` 是干嘛的？"** | Ultralytics 的训练参数，允许输出目录已存在时直接覆写，避免因为上次训练残留目录而报错中断。 |
| **"为什么 AdamW 比 Adam 好？"** | AdamW 把权重衰减（weight decay）和梯度更新解耦了。在 Adam 中，L2 正则化和自适应学习率会互相干扰；AdamW 把 weight decay 单独处理，泛化性能通常更好。 |
